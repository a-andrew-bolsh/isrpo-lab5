# STRATEGIC TEST PLAN

## Проект: Библиотека вычислительной геометрии `geometric_lib`

---

## 1. Концепция и цели тестирования

### Ключевые цели:

* **Верификация алгоритмов:** Подтверждение математической точности функций площади и периметра.
* **Отказоустойчивость:** Проверка поведения системы при получении некорректных типов данных (строки, объекты).
* **Автоматизация контроля (CI):** Внедрение тестов в цикл разработки для мгновенного обнаружения регрессионных ошибок.
* **Валидация типов:** Убедиться, что библиотека корректно обрабатывает преобразование типов или выбрасывает исключения при передаче нечисловых аргументов.

### Приоритетные задачи:

1. Выполнение модульных тестов для геометрических примитивов (круг, квадрат, прямоугольник, треугольник).
2. Проверка точности вычислений при работе с типом `float` (плавающая запятая).
3. **Контроль ввода:** Тестирование реакции функций на строковые аргументы (например, `"5"`, `"abc"`).
4. Сверка результатов с эталонными математическими значениями (константа , формула Герона).

---

## 2. Спецификация программного продукта

### Архитектура:

Библиотека **geometric_lib** построена по модульному принципу, где за каждую фигуру отвечает отдельный файл `.py`.

### Функциональный состав:

* **`circle.py`**: Операции над окружностью (, ).
* **`rectangle.py`**: Вычисления для четырехугольников (, ).
* **`square.py`**: Работа с квадратом (, ).
* **`triangle.py`**: Расчеты для треугольника через три стороны.

### Технический стек:

* **Runtime:** Python 3.10+
* **Dependency:** Модуль `math` для высокоточных вычислений.
* **Interface:** Единый стандарт именования функций `area()` и `perimeter()`.

---

## 3. Объект и границы тестирования

### Элементы, подлежащие проверке:

1. Все публичные методы модулей `circle`, `triangle`, `rectangle`, `square`.
2. **Валидация типов данных:** Проверка поведения функций при получении строк вместо чисел.

### Расширенная область тестирования (New!):

* **Позитивные сценарии:** Передача целых чисел и чисел с плавающей точкой.
* **Отрицательные сценарии:** Передача отрицательных значений (проверка на `ValueError`).
* **Типовая безопасность:** Попытка передачи строковых данных. Система должна либо корректно конвертировать строку в число, либо выбрасывать исключение `TypeError` / `ValueError`.

### Вне рамок (Out of Scope):

* Производительность (Stress testing).
* Графический интерфейс (UI).
* Сетевое взаимодействие.

## 4. Методология и инструменты

### Типология:

* **Unit Testing:** Изолированная проверка каждого метода.
* **Black-Box:** Тестирование на основе ожидаемых результатов без привязки к реализации.

### Инструментарий:

* **Фреймворк:** `unittest` (стандартная библиотека Python).
* **Runner:** Автоматизированный запуск через терминал.

### Сценарии тестирования ввода (Input Validation):

1. **Числовая строка:** Передача `"10"`. Ожидается успешная работа или контролируемая ошибка типа.
2. **Невалидная строка:** Передача `"десять"`. Ожидается выброс `ValueError`.
3. **Пустое значение:** Передача `None` или пустой строки.

### Методы утверждений (Assertions):

* `assertEqual`: Для целочисленных совпадений.
* `assertAlmostEqual`: Для `float` данных (порог точности ).
* `assertRaises`: Для проверки генерации ошибок при подаче строк или отрицательных чисел.

## 5. Регламент приемки (Acceptance Criteria)

Проект признается стабильным при выполнении следующих условий:

1. **Покрытие кода:** 100% функций покрыты как минимум двумя тестами (позитивный + негативный).
2. **Статус тестов:** Все 16+ тестов (включая тесты на строки) имеют статус **PASSED**.
3. **Обработка ошибок:** При вводе строки программа не "падает" с системной ошибкой, а корректно обрабатывает исключение.
4. **CI/CD интеграция:** Успешный прогон тестов на GitHub Actions под Ubuntu и Windows.
